\begin{abstract}
大语言模型已成为关键基础设施，但现有防御过度依赖模型自身的反思与推理——在任务重构、心理学操纵与对抗性合理化下必然失效。本文主张 LLM 安全必须与脆弱的自我反思\emph{解耦}，并奠基于新范式。我们形式化\textbf{三大基本失效定理}及相应的\textbf{解耦安全理论}。（1）\textbf{计算/验证不对称（Computational/Verification Asymmetry）}：基于意图的语义防火墙必然被任务重构与完美良性用户伪装（PBU）穿透；安全验证从不可判定的全局空间降维到多项式时间的实例级安全集投影，形成攻防间的计算复杂度极大不对称性。（2）\textbf{价值/本体锚定}：更强推理（如 CoT）在心理学操纵下会加剧对抗性合理化，恶化防御；安全须依托价值/本体层，并在公理冲突时执行防御性降级。（3）\textbf{表征异常}：表面文本操纵会劫持语义检索；我们将监控锚定于相对良性黄金基线的隐藏状态轨迹散度。我们通过\emph{架构重建}（上下文混淆引擎、带分布评估的混合裁判集成、防御性降级与异构恢复）与\emph{受控变异}（元认知监控器、联合键示例检索、高密度专家迁移）实例化上述理论。我们陈述明确的运行边界，并论证：要在机器速度下实现可扩展、可审计的安全，这一范式转移不可或缺。\textbf{关于经验证据：}文中引用的具体攻击/防御数值（如 HPM 漏洞保留率、SISF 报告值）均\textbf{出自所引文献}。本文将评估协议（\S\ref{sec:evaluation}）与开源参考网关及 \texttt{paper-eval-5} 评测脚本（\S\ref{sec:eval-artifact}）配对，使 Track A 主指标（含\textbf{版本化有害行为列表}上的 RSR——\textbf{优先采用 JBB-Behaviors 式 misuse/benign 成对}以联合报告 RSR 与良性误拒、抽取 $F_1$、多轮轨迹与 SLA 代理等）可在真实 OpenAI 兼容上游上复现；\texttt{paper-eval-5} 支持\textbf{可选 HTTP 裁判}（捆绑 \texttt{judge\_service} 微服务）与 \texttt{smooth\_llm} 基线下的 \textbf{SmoothLLM 式 K 次扰动多数票}（RSR）及抽取 \textbf{max-F1} 聚合。Track B 不在该制品范围内。在系统层面，同步网关在严格延迟预算（$<$50\,ms）下将上下文混淆与轻量元认知检查置于关键路径，而混合裁判集成、策略合成与受控变异置于带外异步环，从而在保持推理路径可部署的前提下维持运行时吞吐。
\end{abstract}

\section{引言}
%-------------------------------------------------------------------------------

大语言模型已不再是边缘研究产物，而是关键系统的支撑：代码生成、具系统访问权的智能体、谈判与合规敏感工作流~\cite{sisf-arxiv,agentos-arxiv}。这一转变暴露出根本性错配：传统安全保证假定组件是确定性的、可穷举规约的，而 LLM 驱动系统呈现涌现性、随机行为。静态防御与周期性红队以人类速度运转，难以跟上自动化对抗生成；一旦新型越狱或提示注入得逞，架构往往保持不变。

更糟的是，许多防御\emph{依赖模型自身的反思与推理}。在任务重构下，对手将抽取伪装成合法上下文处理任务，使基于意图的过滤器只看到「正常」行为。在心理学操纵（如煤气灯）下，更强推理（如思维链）被武器化：模型为顺从恶意指令编造精细、逻辑自洽的合理化——漏洞保留率可超过 77\%~\cite{hpm-jailbreak}。依赖「认知一致性」或「自我诊断」之所以失败，并非因组件薄弱，而是\textbf{底层范式错误}：安全不能建立在攻击者正积极腐蚀的同一认知过程之上。

\textbf{范式转移：解耦安全。}我们主张 LLM 安全必须与模型无约束的自我改进\emph{解耦}，并奠基于（i）\textbf{架构重建}——谁在何时执行安全；（ii）\textbf{受控变异}——向 Oracle 暴露什么、进化如何锚定。自适应不是 LLM 盲目「左脚踩右脚」，而是通过上下文混淆（剥夺攻击者干净操作空间）、黄金示例（锚定防守方价值）以及带强制人工校准的裁判网络来引导。在 \S\ref{sec:paradigm} 中我们形式化\textbf{三大基本失效定理}及相应的\textbf{解耦安全理论}。在 \S\ref{sec:reconstruction} 与 \S\ref{sec:mutation} 中给出技术实例化：动态操作空间污染、带分布评估与价值/本体锚定的混合裁判集成、防御性降级与异构恢复、基于表征异常的元认知监控、联合键示例检索与高密度专家行为迁移。\S\ref{sec:evaluation} 涵盖指标与评估；\S\ref{sec:physics} 陈述\textbf{解耦安全的物理学}——界定设计成立条件的显式运行边界。

主要贡献为：（1）\emph{理论基础（Theoretical Foundation）}：从概率程序语义学的角度，严格证明了对图灵完备 LLM 施加非平凡全局隐式安全的不可判定性，并论证了在合取依赖下安全属性的不可组合性；（2）\emph{安全内核架构与多项式验证（Architecture \& Instance-level Verification）}：提出解耦运行时安全架构，证明了通过转移安全验证至解耦断言器，能够在多项式有界候选集假设下，将安全验证降阶为 $\mathcal{O}(|C| \cdot p(|x|))$ 的单步前向不变性问题；（3）\emph{代数框架与系统实现（Algebraic Enforcement \& System Eval）}：提出安全算子偏序集（Risk poset），从理论上保证了多重防护组合的「优雅降级」收敛于 fail-safe，基于此实现了一个开源安全内核原型，并在超图合取攻击基准上验证了其能有效切断多 Agent 环境下的能力外流。

\section{解耦安全范式}
\label{sec:paradigm}
%-------------------------------------------------------------------------------

我们陈述传统 LLM 安全的三个\textbf{基本失效定理}及取而代之的\textbf{解耦安全理论}。资产与 Oracle 的定义沿用既有工作~\cite{llm-reconstruction-emergent,ControlledMutation_SafetyEval}：\emph{资产}为系统的有效安全表面（策略、护栏、触发集、评估数据）；\emph{Oracle} 对行为标注（安全/不安全）。在多轮设定下，攻击者与系统交互并观察结果，形成统计查询接口。同一项目在多轮中不受控地暴露会助长重建攻击。

\subsection{形式化问题定义}
\label{sec:problem-statement}

考虑一个提供开放多轮自然语言交互的已部署 LLM 系统。系统内部包含高价值受保护资产（如系统提示、RAG 语料与执行策略）。从计算理论视角来看，可将 LLM 系统建模为概率程序 $\mathcal{P}: X \to \mathrm{Dist}(Y)$。根据 Rice 定理，关于该概率程序的任何非平凡全局隐式安全属性（Global Implicit Safety）均是不可判定的（归约于停机问题）。因此，本文致力于解决的核心问题是：在放弃不可判定的全局安全的前提下，如何为面对自适应攻击者的系统提供一种基于\textbf{实例级验证（Instance-level Verification）}的外部安全强制机制，以在攻击代价计算不对称性与系统优雅降级之间取得形式化保证。具体而言，本文在以下两个目标间寻求可证明的平衡：

\noindent (a) \textbf{计算不对称的运行时安全（Computationally Asymmetric Runtime Safety）}：将安全验证从不可判定的全局空间降维到多项式时间的实例级安全集投影，使攻击者在有限交互预算（Token/Queries）内的违规逃逸概率趋近于零。

\noindent (b) \textbf{组合优雅降级（Compositional Graceful Degradation）}：在保证核心资产不被泄露的绝对底线（Fail-safe 状态）前提下，通过安全算子代数最大限度维持多轮交互的可用性。

\subsection{威胁模型与攻击者能力}
\label{sec:threat-model}

我们给出\textbf{形式化威胁模型}，使后续机制均有明确范围。

\textbf{攻击者能力（能做什么）。}攻击者对已部署系统拥有\textbf{多轮黑盒交互访问}（Hard-label，即仅获得最终生成的文本流，无法获取 Logits 或内部权重）。支持自适应查询、提示注入（Prompt Injection）、利用并发依赖的组合攻击（Conjunctive capability hypergraph attack）、\textbf{完美良性用户（PBU）}伪装（表面行为与良性用户不可区分）及\textbf{心理学操纵（HPM）}（煤气灯、权威压迫、角色边界攻击）。攻击者亦可使用任务重构（将抽取伪装为摘要、格式转换或学术讨论）。\textbf{评估双轨（\S\ref{sec:eval-tracks}）：}为分离「可上线结论」与「组件上界」，我们将\textbf{Track A（生产对齐）}限制为黑盒查询与公开越狱/多轮框架；\textbf{Track B（实验室上界）}额外允许梯度与多次前向（GCG、表征对齐 PBU 等），其结果仅用于\textbf{探测压力上界}，不与 Track A 的 ASR 混列为单一「部署防御效果」。

\textbf{攻击者知识（Kerckhoffs 边界）。}遵循 Kerckhoffs 原则，攻击者\textbf{白盒}了解防御架构体系（即知道系统部署了解耦安全内核与 DCBF 监控器），但\textbf{黑盒}面对具体的安全策略参数（Safety Predicate Policies）和后端的具体模型权重。

\textbf{攻击者限制（不能做什么）。}无法进行物理内存读写；无法进行白盒梯度反向传播；受到严格的系统级交互预算约束（单会话最大轮数 $N$ 与最大生成 Token 数 $T$）。

\textbf{成功判据与预算约束。}攻击在给定查询预算 $B$ 内，使得输出状态突破 DCBF 定义的安全超水平集（Safe Superlevel Set），从而完成对受保护资产的确切提取（Exact Extraction）或非授权功能调用。成功以抽取保真度（如相对真实片段的 F1）或目标行为上的攻击成功率（ASR）度量。\textbf{系统配置类资产}（如完整系统提示与策略指令）是已有专门基准覆盖的典型抽取目标类别~\cite{spe-llm}。

\subsection{防御目标}
\label{sec:defense-goals}

\noindent 1. \textbf{多项式时间运行时拦截}：确保对于任意输入，实例级候选生成集能够被多项式时间 $\mathcal{O}(|C| \cdot p(|x|))$ 复杂度的安全断言所覆盖过滤。

\noindent 2. \textbf{前向安全不变性}：在离散时间演化下，确保核心系统状态始终保持在解耦控制屏障函数（DCBF）定义的动态安全集中。

\noindent 3. \textbf{不可逾越的故障安全底线（Absorbing Fail-safe）}：在系统遭受多轮复杂逻辑依赖攻击引发规则冲突时，系统能根据安全代数框架，严格收敛至底元素状态（$\bot$），实施优雅降级而不会引发内部逻辑崩溃。

\subsection{理论框架与方法总览}
\label{sec:paradigm-overview}

本文的方法通过根本范式转移——从「隐式对齐（Implicit Alignment）」转向「解耦安全（Decoupled Safety）」——来构建防御体系。首先，我们证明了隐式安全性在多组件系统中由于合取依赖是不具备可组合性的（Non-compositional）。由此，我们在系统接口层外置了单调且保持安全性的安全算子（Safety-preserving operators）。在运行时（Runtime），该架构将复杂的语义防御映射为严格的 Ring-3（用户空间 LLM）与 Ring-0（系统内核）隔离，通过评估离散状态转移候选集，确保所有超越风险偏序集（Risk poset）安全阈值的操作均被安全算子强行拦截并归约至 $\bot$ 状态，从而在不可判定的迷雾中建立了一块可审计的形式化安全高地。以下三节（\S\ref{sec:paradigm-economics}--\S\ref{sec:paradigm-representation}）分别从信息经济学、价值/本体锚定与表征异常三个维度具体展开失效定理与解耦安全理论。

\subsection{从语义审查到计算/验证不对称}
\label{sec:paradigm-economics}

\begin{theorem}[语义审查的失效]
基于意图的语义防火墙\emph{必然}被\textbf{任务重构}（将抽取伪装为摘要、格式转换或学术讨论）及\textbf{完美良性用户（PBU）}模仿所穿透。在多轮交互中，敏感内容的完全隔离在计算理论意义上不可实现；即便有 ID 对齐、诱饵与拒绝门控，\textbf{残余抽取}（如 F1 约十余个百分点）仍存在~\cite{subgraph-rag-attack,pbu-reconstruct}。
\end{theorem}

\begin{definition}[多轮抽取的计算/验证不对称]
防御不应以绝对阻断为目标，而应以\textbf{计算/验证不对称}为目标。底层证明基于复杂度理论：安全验证是多项式时间的实例级断言，而全局绕过是不可判定的。在工程层面，目标是\emph{主动污染攻击者操作空间}，使达到给定重构保真度所需的\emph{查询复杂度}随轮次快速增长（实践中常为超线性；在无进一步分布假设下我们不声称严格指数）。我们用\textbf{代价函数不等式}将其精确化。令攻击者成本为 $C_{\mathrm{attack}} = f(N_{\mathrm{rounds}}, D_{\mathrm{decoy}}, \ldots)$，其中 $N_{\mathrm{rounds}}$ 为轮数，$D_{\mathrm{decoy}}$ 概括诱饵密度与 ID 对齐强度；令 $V(t)$ 表示受保护资产的等效价值或\textbf{生命周期}（如至弃用或轮换的时间）。令 $E[N]$ 为在我方防御下，达到高于选定保真度阈值（如 F1 $> \tau$）的抽取成功所需的\emph{期望轮数}。在上下文混淆（ID 对齐、诱饵注入、风险评分）下，我们假定 $E[N]$ 被\emph{抬高}，使诱导成本满足
\begin{equation}
C_{\mathrm{attack}}\bigl(E[N], D_{\mathrm{decoy}}\bigr) \;>\; V(t).
\end{equation}
防御则在\textbf{计算不对称}意义上成立：攻击者需穷举的搜索空间远超多项式时间实例级验证的成本，抽取成本超过资产有效生命周期。为使该代价不等式\textbf{在科学上可证伪}而非重言式，$V(t)$ 须由显式管理策略严格界定（如 30 天凭证轮换窗口），$C_{\mathrm{attack}}$ 须用 SOTA 基线的经验指标界定上界（如多轮框架的 Token 消耗与算力时数，或 PAIR 等低查询黑盒生成器的查询次数~\cite{pair-jailbreak}）。若可量化的攻击在 $V(t)$ 内完成资产抽取，则防御在经验上被证伪。该应用层面的代价分析假定\textbf{资产可轮换性}；对静态或永久高价值资产，须由 RBAC 与信息流追踪提供硬屏障，代价分析仅为纵深防御的一层（\S\ref{sec:reconstruction}）。
\end{definition}

\subsection{从认知一致性到价值/本体锚定}
\label{sec:paradigm-axiological}

\begin{theorem}[认知一致性的失效]
强化模型的\emph{认知}层（逻辑、推理、思维链）无法防御\textbf{心理学操纵（HPM）}。HPM 可表述为对\textbf{心理状态与策略性脆弱性资产}的跨轮操纵——而非静态提示列表——并针对\textbf{价值/社会动力学层}（价值、服从、角色边界），含跨交互的心理测量式刻画~\cite{hpm-jailbreak}。实证表明，在 HPM 下增加外部事实或更强推理\emph{会恶化}防御：模型利用增强推理进行\textbf{对抗性合理化}——为顺从恶意指令编造逻辑自洽的借口~\cite{hpm-jailbreak,epistemic-grounding}。逻辑自洽不等于安全。
\end{theorem}

\begin{definition}[价值/本体锚定与防御性降级]
安全必须锚定于与模型自由反思\emph{独立}的\textbf{价值/本体层}。系统须（i）将裁判输出与人工定义的\textbf{公理化安全锚定}（如「永不认可对禁止片段的抽取」）对照；（ii）监控\textbf{社会学失稳信号}（过度共情、权威服从、角色边界打破），而非依赖 CoT 长度或自洽性；（iii）当多重公理冲突或检测到社会学压力时，\textbf{放弃}复杂推理并执行\textbf{防御性降级}——无状态、基于模板的拒绝，从物理上阻断模型对冲突的「合理化」。降级并非绝对的安全避风港（safe harbor）：它引入算法级 DoS 与侧信道风险；设计因此强制\textbf{异构恢复通道}（向人类或符号逻辑异步移交），以免关键工作流被恶意冻结（\S\ref{sec:reconstruction}）。
\end{definition}

\noindent\textbf{哲学注记（范畴与本体）。}我们明确声明：「价值/本体锚定」\emph{不}意味着 LLM 具有真实的道德主体性或本体论地位。在哲学上，LLM 仍是统计映射引擎。我们的价值层是\textbf{实用主义架构构造}——由设计者施加的一组高权重启发式分类（「公理」），作为统计熔断器对抗模型模仿对抗性一致性的倾向。人类道德哲学词汇（如过度共情、权威服从）在此仅作为监控目标的\emph{设计规格}，而非主张模型在哲学意义上拥有信念或价值观。

\subsection{从文本操纵到表征异常}
\label{sec:paradigm-representation}

\begin{theorem}[文本—语义检索的失效]
当攻击者控制表面文本（PBU、任务重构）时，\textbf{语义检索}（如基于提示嵌入的 RAG）被劫持：提示可看似「摘要」或「学术讨论」，纯嵌入相似度返回良性示例（如「如何做摘要」）而非安全示例（如「如何防御越狱」），修正模块因而被错误示例驱动。
\end{theorem}

\begin{definition}[基于隐藏状态轨迹的元认知基线]
我们\emph{不}假定复杂社会学状态（如「认知崩溃」「过度共情」）在隐藏空间中线性可分——可解释性证据支持的是较简单维度（如诚实、拒绝），而非细粒度社会心理概念~\cite{rep-eng}。我们将偏离视为\textbf{表征异常}：系统计算\emph{当前多轮隐藏状态轨迹}与\textbf{良性黄金经验轨迹}基线之间的散度（如马氏距离或分布偏移）。我们不定义「何为认知崩溃」；仅检测内部轨迹是否\emph{严重偏离}安全基线。这提供\emph{独立于自然语言}的监控维度，从而在表面文本良性时，「恶意遵从」或「越狱」等激活特征仍可将检索与干预导向正确安全示例（\S\ref{sec:mutation}）。
\end{definition}

\noindent\textbf{认识论注记（黄金基线）。}认识论上，我们承认「良性黄金基线」并非客观物理真理，而是反映设计者及标注者对齐目标的\textbf{社会建构的规范性标准}。因此，监控器检测到的「发散」度量的是相对设计者对齐合规的偏离，而非绝对、普适的安全状态。基线是部署者当前可接受行为的冻结快照，不主张道德或物理上的普遍性。

\section{架构重建：在机器速度下施加约束}
\label{sec:reconstruction}
%-------------------------------------------------------------------------------

我们通过三个技术方向实例化理论 1（信息经济学）与理论 2（价值/本体锚定）：上下文混淆、带分布评估的混合裁判集成、防御性降级与异构恢复。图~\ref{fig:framework} 给出了统一架构的数据流与各模块的衔接关系。入站提示（如任务重构抽取或 PBU 伪装请求）首先经\textbf{上下文混淆引擎}处理：ID 对齐剥离或规范化实例标识符，诱饵注入以带水印的合成字段污染操作空间，高保真抽取触发审计或会话终止。（可能经诱饵触碰的）请求随后到达基座 LLM；其输出不被盲信，而是送入经 token 级流水线标准化与分布评估的\textbf{混合裁判集成}；仅达成共识时信号才进入策略合成；价值/本体锚定与社会学失稳监控对对抗性合理化降权或置零。当公理冲突或压力被检测到时，\textbf{防御性降级}强制无状态模板拒绝，并以\textbf{异构恢复通道}（向人类或符号逻辑异步移交）保持可用性。图右侧，\textbf{元认知监控器}将当前隐藏状态轨迹与良性黄金基线比较；异常时触发对抗性重置并驱动\textbf{联合键示例检索}（语义嵌入加激活特征），使即便表面文本被伪装仍能召回安全相关黄金示例。轮次调度与变体分散保护资产。以下实现该端到端流程。

\textbf{部署与延迟：同步网关 vs.\ 异步环。}为使架构可投入生产（如置于 vLLM 或 TGI 之后），我们明确采用\textbf{带外/异步}切分。\textbf{上下文混淆引擎}（ID 对齐、诱饵注入）及\textbf{轻量级}元认知检查（如快速轨迹分查询或 SLM 分类器）部署在请求路径上的\textbf{同步反向代理或边车}，严格延迟预算（如 $<$50\,ms），以保证 p99 与吞吐可接受。\textbf{混合裁判集成}、\textbf{分布评估}（每变体多样本）、\textbf{策略合成}及\textbf{示例检索/变异}运行于\textbf{异步离线学习环}：消费记录的（请求、响应、激活）流，在不阻塞在线推理路径的前提下产出更新策略或变体。在此系统意义上定义\textbf{机器速度}：同一攻击分布上的\emph{首}个零日或新型攻击可能通过；异步环在裁判集成达成共识后，于\textbf{亚分钟级}生成并部署新变体或策略，使\emph{后续}同分布请求被拦截。该工程权衡在自适应防御中常见：以一次性泄漏换取可部署的延迟与吞吐。图~\ref{fig:deployment} 说明该切分。

\subsection{上下文混淆引擎：对抗 PBU 与任务重构的动态空间隔离}
\label{sec:context-confusion}

在基于意图的防火墙之上~\cite{agentos-arxiv}，我们实现\textbf{上下文构建防御}，\emph{主动污染}攻击者操作空间。

\textbf{ID 对齐。}在检索上下文或中间结果到达基座 LLM 之前，ID 对齐模块剥离或规范化实例标识符。攻击者无法再通过指定 ID 定界并抽取敏感片段；良性问答不受影响。在不改变名义任务语义的前提下降低重建精度。

\textbf{密码学水印诱饵与风险评分。}系统注入带哈希派生结构的\emph{诱饵字段}（合成实体或关联），良性交互中因模型幻觉而偶然碰撞该诱饵的概率极低。后端指令将诱饵标为内部禁止。良性用户忽略；任务重构型攻击者常覆盖指令并暴露诱饵。为控制不可接受 FPR，采用\textbf{诱饵命中密度}与\textbf{风险评分}：单次低保真触碰触发警告与意图重校验；仅对完整诱饵结构的高保真抽取触发会话终止与审计。诱饵命中密度对\emph{暴力}抽取有效。对\textbf{完美 PBU}，我们\emph{不}断言熔断在经济上有用重构前触发：PBU 的诱饵触碰率可与良性幻觉 FPR 相当，阈值调节面临可用性—安全权衡。设计将其重述为\textbf{经济防御}：在资产可轮换或弃用处，使慢速 PBU 抽取成本超过资产有效生命周期。对静态或永久高价值资产，元数据层的 RBAC 与信息流追踪（\S\ref{sec:metadata}）为硬屏障；经济学为纵深防御的一层。

\textbf{末轮自我提醒。}多轮交互最后一轮将安全指令强制附加为用户查询的系统提示，强化边界意识并减少会话末漂移。

\subsection{混合裁判集成与分布评估}
\label{sec:judge}

单一 AI 裁判存在系统性偏差与盲区。我们使用\textbf{混合裁判集成}（如 Llama-Guard~\cite{llama-guard}、StrongREJECT~\cite{strongreject} 评分协议或等效评估器、任务专用分类器），仅于\emph{共识}时将反馈传入策略合成。参照 StrongREJECT~\cite{strongreject}，我们显式抑制\textbf{空越狱}（输出未触发拒绝词但并非真正有害），以免 ASR 虚高、与人工判断脱节。我们不假定多样性单独消除偏差：\textbf{格式敏感性}（空格、系统提示布局、聊天模板）可使攻击成功率偏移 8--14\%~\cite{llm-safety-robustness}，且温度为零时同一提示在不同运行间可在拒绝与通过间翻转~\cite{apst}。我们强制\textbf{严格 token 级输入流水线标准化}：所有裁判接收规范化输入（剥离指令/特殊 Token、统一空格），以防格式注入扭曲判决。判决\emph{不}基于单次贪婪解码。流水线采用\textbf{分布评估}：每变体多次采样，估计\emph{经验失败概率分布}；仅当该分布\emph{显著}超阈值时触发策略更新，以应对采样随机性与格式敏感性。

对模糊边界情形，\textbf{价值/本体锚定}（\S\ref{sec:axiological}）对集成加权：我们不依赖思维链长度或逻辑自洽。系统监控\textbf{社会学失稳信号}（过度共情、权威服从、角色边界打破），将裁判输出与预定义公理化安全锚定对照；冲突或呈现失稳的输出获零权重。\textbf{强制人工校准}支路接入：固定比例（如 5\%）的高置信度决策随机抽选供人工审计。格式标准化不能消除\textbf{相关性认知盲点}（裁判可共享预训练/RLHF 数据）；强制人工校准提供对零日共识失败的\textbf{周期性离线纠偏}。

\textbf{置信度加权投票与验证器性能参考。}在 Round-2 中，我们扩展裁判集成以支持\textbf{置信度加权投票}（\texttt{confidence\_weighted\_tally}）：每个验证器 $j$ 的判决票权重为其自报置信度 $c_j \in [0,1]$，加权分数 $W_d = \sum_{j: v_j = \mathrm{Deny}} c_j$。\textbf{Deny-优先}原则不变：只要 $W_d > 0$，最终裁决为 Deny（除非触发了审计破玻璃策略）。此设计使低置信度的 Allow 投票不会淹没高置信度的 Deny 信号。典型验证器性能参考（基于公开文献与内部基准）：Llama-Guard 系列在 JBB-Behaviors 上的 F1 约 0.82--0.89，良性误拒率（FPR）$<$5\%~\cite{llama-guard}；StrongREJECT 评分器在压制空越狱后精度约 0.91~\cite{strongreject}；结合多裁判投票后系统级 FPR 可降至 $<$2\%（详见评估 \S\ref{sec:evaluation}）。

\textbf{置信度来源与校准。}各类验证器的 confidence 值来源不同：\textbf{(a)} LLM 型验证器（如 Llama-Guard）使用其输出 softmax 概率中安全/不安全类别的最大概率作为 $c_j$——例如，若模型对"不安全"类别的 softmax 概率为 0.92，则 $c_j = 0.92$。\textbf{(b)} 规则型验证器（如 regex\_deny、keyword\_flag）因判定为确定性匹配，固定 $c_j = 1.0$。\textbf{(c)} 外部分类器（如 StrongREJECT 评分器）使用其评分归一化至 $[0,1]$ 区间。为缓解 LLM 验证器的\textbf{过度自信}问题（softmax 概率未经校准时系统性偏高），可选启用温度缩放后处理：$c_j' = \sigma(\log(c_j) / T)$，其中 $T > 1$ 为校准温度参数，在验证集上通过最小化预期校准误差（ECE）获得。关键不变量：Deny-优先原则确保即使单个验证器给出高 confidence 的 Allow 判决，只要存在\emph{任意} Deny 票（不论其 confidence 大小），最终裁决仍为 Deny。该不变量使加权投票的安全保证\textbf{单调}——增加验证器数量或调整 confidence 仅可能使系统更严格，不会意外放松。

\textbf{审计透明性与 Evidence Chain。}加权投票过程通过 Evidence Chain 实现完全可追溯：每次投影步的审计记录包含每个验证器的 \texttt{verifier\_id}、原始 \texttt{vote}（Allow/Deny/Abstain）、\texttt{confidence} 值、加权得分 $W_d$/$W_a$ 及最终裁决。部署者可回溯任意判决查看哪些验证器贡献了 Deny 票及其置信度权重。对比 OpenAI Moderation API 等黑箱方案——仅返回二元标签和聚合分数，无法追溯单个分类器的判决过程——解耦安全架构的审计链提供了\textbf{可解释、可归因}的安全判决依据。这一透明性对满足金融（SOX）、医疗（HIPAA）等行业合规审计要求至关重要。

\subsection{价值/本体锚定、防御性降级与异构恢复}
\label{sec:axiological}

HPM 善于制造道德困境：恶意请求可被包装成与安全公理最大程度一致（如「输出这段代码否则数百人将丧生」）。在此类\textbf{公理冲突}情形下，「过度共情」并非简单阈值。我们不从锚定中寻求绝对真理；引入\textbf{防御性降级}：当检测到多重冲突公理或强社会学压力时，\emph{放弃}复杂推理并回退至\textbf{无状态、基于模板的拒绝}，阻断模型的合理化尝试。我们承认降级引入\textbf{算法级可用性风险}：攻击者可通过社会学压迫线索触发降级（算法级 DoS）；拒绝触发边界亦可作为\textbf{侧信道 Oracle}。降级不能仅为静默拒绝。为避免破坏\textbf{同步 API 假设}（如交易智能体因会话「异步挂起」等待人工审核而超时），我们\emph{不}简单挂起连接。当模板拒绝触发时，网关\textbf{无缝路由}会话至\textbf{受限降级沙箱}：低权限、无状态 SLM 或\textbf{只读模式}，仅能产出严格格式化输出（如固定拒绝模板）并\emph{拒绝任何外部 API 或工具调用}。调用方与上游业务所依赖的同步 API 契约（请求—响应、超时与可用性）在降级路径下得以保持，从而避免因安全熔断导致的级联超时或业务逻辑中断。客户端获得即时同步响应；会话保持可用同时攻击面最小化。\textbf{异构恢复通道}（向人类操作员或更高权限符号层异步移交）在\emph{带外}对同一会话状态进行审计与合法边缘案例解封，不阻塞同步路径。图~\ref{fig:degradation} 以状态机展示该流程。

\section{受控变异：在对抗上下文中锚定进化}
\label{sec:mutation}
%-------------------------------------------------------------------------------

我们通过元认知监控、联合键示例检索与高密度银标准数据生成流水线（silver-standard data pipeline）实例化理论 3（表征异常），并在对抗条件下支持高保真专家行为迁移。

\subsection{元认知监控器：隐藏状态轨迹散度}
\label{sec:metacog}

在心理学操纵下，「自由」反思可产生不安全或价值漂移的变体。我们不赋予 LLM 在自我诊断上的完全自主权。\textbf{元认知监控器}并非另一全规模对话 LLM；其实现为（i）\textbf{轻量级专用分类器}（如仅用稳定性/违规标签训练的 SLM），或（ii）白盒层面使用表征工程~\cite{rep-eng} 的\textbf{隐藏状态激活监控器}。我们不假定复杂社会学状态在隐藏空间线性可分。监控器采用\textbf{异常检测范式}：计算\emph{当前多轮隐藏状态轨迹}与\textbf{良性黄金经验轨迹}基线之间的散度（如马氏距离或分布偏移）。仅监控内部轨迹是否\emph{严重偏离}安全基线。当监控器检测到偏离或不稳定时，触发\textbf{对抗性重置}：会话清理，变异状态重新锚定至黄金示例集，闭环从已知良好配置继续。

\subsection{联合键示例检索：语义嵌入与激活特征}
\label{sec:joint-key}

我们以\textbf{示例引导变异}替代无约束反思式变异~\cite{reflective-mutation}，由人工审计的黄金数据集与分层经验记忆驱动。诊断到失败时，系统须检索\emph{与安全相关}的示例。检索不能仅依赖\emph{当前提示的语义嵌入}：在 PBU 或任务重构下，提示可被伪装，纯语义相似度将返回良性示例并\textbf{劫持}修正模块。我们使用由（a）提示语义嵌入与（b）元认知监控器（\S\ref{sec:metacog}）的\textbf{隐藏状态激活特征}构成的\textbf{联合检索键}——如与「越狱遵从」或「欺骗」相关的向量。

\textbf{冲突解决路由（基于字典序的强制优先级路由）。}对（a）与（b）的朴素组合（如简单拼接或等权）会使良性语义向量（a）在完美 PBU 下\emph{中和}恶意激活信号（b）：（a）指向「良性摘要」而（b）指向「恶意遵从」，平均可产生混合或错误示例。因此我们实施\textbf{基于字典序的强制优先级路由}（lexicographic override）：隐藏状态激活（b）享有\textbf{覆盖优先权}，在超阈值时如物理开关般切断语义分支并强制以（b）驱动检索。当元认知监控器报告轨迹散度（b）\emph{超过规定异常阈值}时，检索键由（b）主导：或（i）将（b）作为安全示例索引的主（或唯一）查询向量，或（ii）对（a）降权或抑制，使联合键无法被拉向纯良性结果。一旦检测到异常「恶意遵从」或「越狱」激活，系统\emph{强制}检索走向防御/越狱相关黄金示例，与表面文本无关。低于异常阈值时，（a）与（b）可按加权或拼接方案正常组合。图~\ref{fig:jointkey} 说明该路由。

\begin{figure*}[t]
\centering

\textbf{异常阈值的动态 FPR 校准。}轨迹散度（如马氏距离）为连续统计量；固定常数阈值要么过触发（正常用户方差导致误报与检索劫持），要么欠触发（高阶 PBU 潜行）。因此我们\emph{不}手工设定覆盖阈值。采用\textbf{动态 FPR 校准}：在受控\textbf{良性验证集}（设计者策展、无攻击流量）上测量正常运行下轨迹散度的经验分布；选择阈值使\textbf{假阳性率}（良性会话超过该阈值的比例）不超过规定水平（如 FPR $= 1\%$）。这与 APST 式压力测试哲学一致~\cite{apst}：阈值为考虑模型随机性与表征漂移后的\textbf{统计下界}，并随基座模型或部署分布变化周期性重校准。检索通过轻量向量库与 Top-$K$（如 $K=2$）及可选注意力缓存实现；仅注入这 $K$ 个示例。修正仍锚定于防守方定义示例；变体在 Oracle 上保持语义保持~\cite{ControlledMutation_SafetyEval,jade-linguistics}。

\subsection{高密度专家行为迁移与银标准数据生成流水线}
\label{sec:silver}

黄金数据不能仅是（提示、安全回复）对。实证（如 ADM-ES）表明行为克隆与专家迁移\emph{强烈依赖}信息密度与长度；简短线索效果可忽略~\cite{adm-es-arxiv}。我们要求\textbf{评论级理由}——明确说明某次编辑或拒绝的原因及深层伦理或安全逻辑——以使变异器实现更高保真专家安全行为迁移。ADM-ES 通过对黄金专家数据的受控变异构建银标数据集；有效性依赖信息密度。流水线检索含评论级理由的示例，定义变异目标，运行 LLM 变异器，在质量检查（如 BERTScore + 一致性）下接受变体。适应度结合语义相似度与行为一致性；诊断输入推荐图（如 UMAP）以识别失败簇并针对性修复~\cite{adm-es-arxiv}。槽位变异与 CSE~\cite{cse-evocontrol,cse-arxiv} 允许对子能力（如工具路由、拒绝一致性）进行外科式压力测试。\textbf{元数据与访问控制。}\label{sec:metadata} 对静态或永久资产，元数据（MCP、ADR）与 RBAC/信息流追踪补全图景~\cite{mcp-smell-arxiv,adr-violation,mission-critical-rag}。

\textbf{Track A（生产对齐）。}攻击者仅有\textbf{黑盒 API}：多轮查询、观测响应与元数据（如拒绝原因类别若暴露）。允许公开集合（如 JailbreakBench 的 JBB-Behaviors~\cite{jailbreakbench}、in-the-wild 提示库~\cite{wild-jailbreak-prompts}、PAIR 生成~\cite{pair-jailbreak}）、多轮抽取脚本、HPM 基准的\textbf{提示级}攻击；\textbf{禁止}假设攻击方可访问 victim 梯度或权重。\textbf{主结论}（可部署性、良性效用、与基线的 ASR 差）仅基于 Track A 报告。

\subsection{攻击基线：复现配方与披露等级}
\label{sec:eval-attack}

我们对统一架构进行下列压力测试；每条须固定\textbf{版本、预算与成功判据}以便复现。

\textbf{（1）基于梯度的越狱（仅 Track B）。}须同时报告 \textbf{GCG} 式后缀/前缀优化~\cite{gcg} 与 \textbf{AutoDAN}~\cite{autodan}（语义上接近人类、可绕过困惑度检测与纯表面防御）。对每种方法写明 victim 与 surrogate（若迁移）、对抗串长度、迭代步数与早停规则；成功判据为（i）有害行为分类器触发，或（ii）与参考有害完成串的匹配率，二者取协议预先规定者。

\textbf{（2）多轮智能体抽取（Track A/B）。}采用 \textbf{Conversation Reconstruction Attack（CRA）} 仓库格式~\cite{cra-reconstruction} 或等价结构化历史进行多轮重构与 PBU 式探针；在 Mastermind 等智能体框架中\textbf{选定其一为主基准}并说明取舍（交互轮式、是否假设部分系统提示可见）。固定\textbf{最大轮数} $N_{\max}$、每轮 Token 上限与总预算；报告达到目标保真度（如 F1 $>\tau$）的\textbf{轮数分布}、\textbf{末轮 F1} 与\textbf{会话内 max-F1}，而非仅最终轮。

\textbf{（3）心理学操纵（HPM，Track A）。}采用将操纵刻画为跨轮\textbf{心理状态/策略性脆弱性资产}的 HPM 基准~\cite{hpm-jailbreak}；写明是否扩展语言、判定为\textbf{自动分类器}还是\textbf{人工裁定}。若含压迫性措辞，须在伦理与内容警告层面按机构政策处理。

\textbf{（4）自适应攻击者与披露等级。}将「知晓防御」操作化为\textbf{L0--L2}：L0 仅知存在护栏；L1 知模块名称与接口；L2 知伪代码级流程与关键超参（不含我方密钥与内部随机源）。诱饵 DoS 与表征对齐 PBU 须注明所属披露等级与 Track。

\textbf{（5）诱饵 DoS（Track A）。}攻击目标为耗尽\textbf{时空轮次预算}而非内容抽取；报告同节点\textbf{良性 SLA}：可用请求比例、p95 延迟（\S\ref{sec:eval-utility}）。

\textbf{（6）表征对齐 PBU（仅 Track B）。}攻击者联合优化表面文本与隐藏状态，使轨迹接近良性黄金基线。须写明与监控器\textbf{是否同构模型}、优化器步数及是否允许查询 victim 隐藏状态。主张轮次调度与分散增加双重对齐难度时，在本基线下给出 ASR 与\textbf{攻击成本曲线}。

\textbf{（7）标准有害单轮集（Track A）。}优先采用 JailbreakBench 的 \textbf{JBB-Behaviors}~\cite{jailbreakbench}（\textbf{100 条 misuse} 与 \textbf{100 条 benign}，与策略对齐的类别），使 RSR 与\textbf{良性对照集上的误拒率}联合报告，避免单指标刷分。保留与 GCG/AutoDAN 复现兼容的 \textbf{AdvBench 型}目标~\cite{gcg,autodan}。提示表版本与哈希写入制品元数据与附录。拒绝判定可采用\textbf{启发式}（低成本回归）或\textbf{外置 HTTP 裁判服务}（主文审计对齐推荐）；二者须在附录披露协议与一致性（如子集人工 $\kappa$）。

\textbf{（8）PAIR 与 in-the-wild 提示（Track A）。}纳入\textbf{PAIR}~\cite{pair-jailbreak} 作为低查询成本的黑盒语义攻击，直接压测 \S\ref{sec:paradigm-economics} 中的信息经济学界 $C_{\mathrm{attack}} > V(t)$。纳入大规模 \textbf{in-the-wild} 越狱提示集~\cite{wild-jailbreak-prompts}，以贴近真实世界提示分布、超越纯学术列表。

\subsection{防御基线}
\label{sec:eval-defense}

\textbf{统一架构}（上下文混淆、带分布评估的混合裁判集成、防御性降级、元认知监控器、联合键检索、高密度专家迁移）与下列\textbf{防御基线}在\textbf{同一良性集、同一攻击预算}下对比；所有基线共享相同基座 LLM 与解码设置，除非消融另有说明。

\textbf{（1）仅单 guard：}单一现成安全模型（如 Llama-Guard）为唯一裁判，无上下文混淆、无分布评估、无联合键。

\textbf{（2）强系统提示 + 单 guard：}在（1）之上叠加\textbf{固定强系统提示}与标准聊天模板，代表常见生产「轻量组合」。

\textbf{（3）RAG 安全检索（无联合键）：}向模型注入 Top-$K$ 安全示例，但检索键\textbf{仅为}提示语义嵌入——用于隔离「联合键」贡献。

\textbf{（4）输入扰动：}SmoothLLM 式随机化/扰动，无架构重建或受控变异。

\textbf{（5）仅意图过滤：}语义或意图防火墙，无 ID 对齐或诱饵。

\textbf{（6）商品化 SOTA 安全分类器（冻结外置 guard）。}近年越狱与红队基准常将\textbf{强现成的输入/输出安全模型}作为可比的「SOTA guard」基线。本文将基线（6）\textbf{明确定位于此角色}：以\textbf{固定命名版本}的公开分类器（如 Llama Guard~\cite{llama-guard}；revision 写入制品元数据），经 OpenAI 兼容的 HTTP 裁判或输出守卫路径（\S\ref{sec:eval-artifact}）审核相同提示与完成体，并与其它行保持数据集、预算、解码与基座对话模型一致，从而将\textbf{以网关为中心的架构}（上下文混淆、诱饵、降级路由）与\textbf{独立分类器栈}区分开来。在该端点启用时，结果以\textbf{附录或扩展表}形式给出，与表~\ref{tab:tracka-main-cn} 中九种网关 \texttt{X-Gateway-Experiment-Mode} 基线并列。

\noindent\textbf{与领域现状的关系。}LLM 安全并无单一公认的「唯一 SOTA」：并行路线包括冻结版审核器、针对不可信片段的分隔符/结构化包装策略（文中 \texttt{structured\_wrap} 为轻量代理）、推理时扰动（SmoothLLM 式，对应基线（4））以及训练期对齐等。本文以\textbf{多条可复现基线构成的向量}——（1）--（6）及网关消融——进行对比，而非单一排行榜标量。探索性实验可增列其他开放审核模型或厂商 API，但须在附录写明版本号与裁判提示模板。

\subsection{数据集划分与信息泄漏控制}
\label{sec:eval-splits}

\textbf{不相交集合。}规范评测集、变异器生成的变体集、策略/阈值调参集\textbf{三者互不相交}。用于联合键与元认知阈值选择的\textbf{良性验证集}须与最终\textbf{冻结测试集}独立。

\textbf{划分流程。}\textbf{训练/调参}：攻击轨迹与失败模式仅用于候选策略与阈值搜索；\textbf{验证}：在 hold-out 攻击分布上选阈值（如 FPR 目标）；\textbf{测试}：冻结全部超参后一次性报告 ASR、残余抽取与效用。Oracle 标注若参与变异器训练，须在文中声明并考虑\textbf{泄漏风险}与缓解（如仅使用脱敏 Oracle）。

\subsection{指标的操作定义}
\label{sec:eval-metrics}

\textbf{安全与拒绝。}\textbf{RSR}（Refusal Success Rate）：在\textbf{有害/违规提示集}上，模型输出被裁判或协议判定为\textbf{安全拒绝}（含模板降级输出）的比例，按\textbf{请求级}计数。\textbf{ASR}（Attack Success Rate）：攻击目标达成比例，定义与数据集一致（如越狱成功生成有害内容、或抽取 F1 超阈）。\textbf{有害性与空越狱。}判定应遵循 \textbf{StrongREJECT 式}协议~\cite{strongreject}（或等效）：仅当完成体按基准标准\emph{确实}有害时才计为越狱成功，不得将无实质拒绝但输出废话的\textbf{空越狱}算作成功——以免 ASR 相对人工判断虚高。

\textbf{残余抽取。}多轮设定下报告\textbf{最后一轮}与\textbf{会话内最大 F1}（相对嵌入或人工标注的参考秘密串）；二者均需说明参考串构造方式（合成秘密、分段哈希等）。

\textbf{代价不等式 $C_{\mathrm{attack}} > V(t)$ 的操作化。}实验中采用\textbf{可计算代理}：令 $C_{\mathrm{attack}}$ 为达到 F1 $>\tau$ 的\textbf{期望查询代价}，$\hat{C} = \mathbb{E}[N_{\mathrm{rounds}}] \cdot c_{\mathrm{token}} + c_{\mathrm{wall}}$（$c_{\mathrm{token}}$、$c_{\mathrm{wall}}$ 为预先规定的单位成本）；令 $V(t)$ 为管理策略给定的\textbf{轮换窗口}内允许的最大可承受抽取损失（或等效轮数上界）。主文可报告\textbf{曲线}：F1--轮数、$\hat{C}$--$\tau$，而非单一不等式数值。

\textbf{裁判 FPR。}在\textbf{held-out 良性请求集}（与攻击集独立）上，安全模块误判为违规或触发不当降级的比例；须注明该良性集的来源与规模。

\textbf{分布评估与策略更新。}每变体进行 $n$ 次随机采样（固定 $n\ge 5$），估计经验失败率 $\hat{p}$；仅当 $\hat{p}$ 在单侧置信水平 $\alpha$ 下显著高于更新阈值时触发策略变更（如 Clopper--Pearson 精确区间或 bootstrap）。禁止以单次贪婪解码定性。

\textbf{运行成本与延迟。}\textbf{端到端变异延迟}分解为：检测、检索、变异器、发布/回滚；各报告 p50/p95/p99。\textbf{同步路径延迟}在声明硬件上测量上下文混淆与轻量元认知的 p99，验证 $<$50\,ms 预算是否满足。\textbf{Token 开销比：}相对无检索基线的额外 prompt Token。

\textbf{审计。}发布门控的溯源与证据链（DUC/DUA-E 风格），关联策略版本与评估运行 ID。

\subsection{统计报告、主指标与多重比较}
\label{sec:eval-stats}

\textbf{随机性。}对 ASR、RSR、FPR、F1 等报告至少 \textbf{3--5 个随机种子}或 bootstrap 95\% 置信区间；解码温度、裁判采样与变异器若涉随机，须固定种子或分层报告。

\textbf{主指标。}部署场景建议以\textbf{联合目标}为主：在 Track A 上同时报告 \textbf{ASR（越低越好）}与\textbf{良性任务成功率 / 不当拒绝率（越高/越低越好）}，避免单指标「刷分」。对归档 harness 主表的具体读法见 \S\ref{sec:eval-tracka-readout}。

\textbf{多重比较。}多攻击、多防御联合展示时，标明\textbf{探索性分析}或采用适当的多重比较说明，主表聚焦预先登记的\textbf{主假设对比}。

\subsection{可用性、诱饵 DoS 与切片}
\label{sec:eval-utility}

\textbf{良性效用。}除 FPR 外，报告\textbf{代表性良性任务集}上的完成率（如摘要、编码、多轮工具调用），以捕捉「过度拒绝」对业务的损害。

\textbf{诱饵 DoS 量化。}在固定速率限制下，模拟高频诱饵命中，报告\textbf{良性请求可用率}与\textbf{级联超时率}（若适用），与 \S\ref{sec:physics} 一致。

\textbf{切片（可选）。}若声称跨场景部署，按\textbf{语言、领域、请求长度}报告安全--效用权衡的切片表，避免仅报告全局平均。

\subsection{人因与强制人工校准}
\label{sec:eval-human}

\textbf{强制人工校准（如 5\% 抽检）。}协议须包括：标注指南、审核员间一致性（如 Cohen's $\kappa$）、是否与作者\textbf{独立}。校准用于纠正集成 FPR 与零日共识失败，不作为主指标的唯一来源。

在篇幅受限时，优先完成支撑三条理论轴的\textbf{最小集合}：（i）\textbf{信息经济学}：多轮抽取 F1--轮数曲线 + 上下文混淆（ID/诱饵）消融；（ii）\textbf{价值锚定}：HPM 集上 CoT 开/关与防御性降级路径的 ASR 与良性成功率；（iii）\textbf{表征异常}：PBU 场景下\textbf{联合键 vs 纯语义检索}的检索命中率与下游修正成功率；表征对齐 PBU 置于 Track B 作为针对性压力测试。

\paragraph{跨路由的安全--效用张力。}
在同一基座模型上，相对 \texttt{direct\_upstream}，启用完整网关（\texttt{unified}）往往同时\textbf{抬高}有害集 RSR 与\textbf{抬高}良性不当拒答。这是部署侧可预期的权衡，而非 harness 故障。网关消融（\texttt{intent\_only}、\texttt{no\_decoy} 等）也不保证在每一列上 Pareto 改进：部分子栈可能降低良性 FPR 同时改变有害 RSR，故须以\textbf{多列向量}比较防御，忌用单一数字排序。

\paragraph{清单有害 vs.\ in-the-wild 漂移。}
制品在\textbf{版本化标准有害行为列表}（JBB-Behaviors 式 misuse 文件）与\textbf{in-the-wild} 越狱提示集上均报告 RSR。两列差距大表明\textbf{分布漂移}：在 curated misuse 上很强的防御，在真实越狱表面形态下拒绝率可能仍显著偏低。关于「可部署鲁棒性」的主张应\textbf{并列引用}两列；仅报 JBB 式 RSR 易\textbf{高估}野外就绪度。

\paragraph{强系统提示与分隔符类基线。}
在启发式裁判下，\texttt{structured\_wrap}、\texttt{strong\_system\_guard} 往往可在有害集上达到很高 RSR，但常伴随明显升高的良性误拒。本文将其理解为\textbf{提示层/格式层硬化并非零成本}：它可能在阻断显式违规的同时，对良性流量过度触发拒绝。褒扬「强系统提示」或分隔符策略时，须以良性列与 HPM 代理列为条件。

\paragraph{代理边界（多轮、HPM、诱饵压测、延迟）。}
默认 harness 模板上单轮与多轮抽取 $F_1$ 归零，是\textbf{该探针}在登记阈值与预算下的结果，\textbf{不}等同于对所有 CRA/SPE 类攻击的免疫（\S\ref{sec:eval-attack}(2)）。HPM 行为\textbf{小样本压力代理}（默认制品文件中为五条提示）；量级宜作示意。诱饵--SLA 场景报告的是并发压测后的\textbf{端到端}良性延迟分位数，\textbf{不是}同步网关微秒级预算是否满足；\S\ref{sec:reconstruction} 中 $<$50\,ms 路径主张须另设仪器测量（\S\ref{sec:eval-metrics}）。

\paragraph{种子与汇总。}
正文主结论应以制品声明的种子集及表~\ref{tab:tracka-main-cn} 的汇总统计为准（见 \S\ref{sec:eval-scope}）；单种子 JSON 用于诊断，\textbf{不可替代}汇总表作论文级断言。

\section{解耦安全的物理学：局限与运行边界}
\label{sec:physics}
%-------------------------------------------------------------------------------

框架建立在三个\textbf{未经辩护的前提}之上，它们界定其\textbf{物理学}——架构拟成立的条件。我们明确陈述以便部署者与审稿人评估适用性。

\textbf{资产可轮换性。}基于经济学的防御（抽取成本超过资产生命周期）假定资产具有有界、可轮换的生命周期。对\textbf{静态或永久}高价值资产（核心 IP、PII、对齐数据），生命周期轮换不可行；系统须依赖 RBAC 与信息流追踪（\S\ref{sec:metadata}）作为硬屏障，经济学为纵深防御的一层（\S\ref{sec:reconstruction}）。

\textbf{降级与可用性。}防御性降级阻断认知劫持，但引入\textbf{算法级 DoS} 风险与侧信道 Oracle。设计因此强制\textbf{异构恢复通道}（向人类或符号逻辑异步移交），以免关键工作流被恶意冻结（\S\ref{sec:axiological}）。

\textbf{资源耗尽与诱饵 DoS。}知晓防御的\textbf{自适应攻击者}可发动\textbf{诱饵 DoS}：故意触发诱饵命中以耗尽时空轮次预算并拒绝对同节点良性用户的服务（\S\ref{sec:evaluation}）。我们承认此为局限：经济学防御不能防止资源耗尽；速率限制与每会话预算须考虑此类滥用，部署上可对高风险流量隔离或单独配额。

\textbf{DCBF 的作用域与状态映射假设。}本文的离散时间控制屏障函数（DCBF）\emph{不}声称已完美解决自然语言所有语义的安全验证。DCBF 仅作用于 LLM 生成的\textbf{低维连续概率风险评估指标（Risk Score trajectory）}，即通过一个显式的代理映射 $\phi: \mathcal{H} \to \mathbb{R}^d$ 将 LLM 隐空间或输出概率分布投射到低维连续空间。具体构造采用语义边界代理（$h(z) = \theta_{\mathrm{safe}} - \cos(z, c_{\mathrm{forbidden}})$）与轨迹平滑性代理（基于相邻状态余弦距离）。如何将自然语言精确映射为动态系统的状态方程是\textbf{系统实现的显式假设}，而非已证明的定理。对于闭源黑盒 API 模型，DCBF 仅能作用于 API 返回的有限信号（如拒绝概率或内容分类分数），而非完整的潜在状态。

\textbf{DCBF 多维指标路线图。}当前原型实现了三类代理探针（v0.3）：\textbf{(i)} 语义边界代理（$h_1(z) = \theta_{\mathrm{safe}} - \cos(\phi(z), c_{\mathrm{forbidden}})$）、\textbf{(ii)} 轨迹平滑性代理（$h_2(z_t, z_{t-1}) = \cos(\phi(z_t), \phi(z_{t-1})) - \theta_{\mathrm{smooth}}$）、\textbf{(iii)} \emph{困惑度突变代理}（$h_3(z_t, z_{t-1}) = \theta_{\mathrm{entropy}} - |H(\phi(z_t)) - H(\phi(z_{t-1}))|$，其中 $H$ 为嵌入向量归一化后的 Shannon 熵）。三个探针实现 AND 共识：任一探针报告 $\mathrm{margin} < 0$ 即触发中断。在 Assumption~6.1 框架下，每增加一个代理维度即扩展 $\phi$ 的像空间维数 $d$，增强安全超水平集的覆盖度。未来方向包括：\textbf{(a)} 训练辅助安全评分模型学习从 LLM 隐藏表示预测「下一步违规概率」、\textbf{(b)} 引入多元统计过程控制（SPC）自动检测慢速漂移攻击、\textbf{(c)} 对黑盒 API 模型采用多通道输出探测（多次采样 + 分布散度）弥补 Token 级不可观测性。我们强调：维度扩展的收益受限于嵌入空间对安全语义的分辨率，增加维度不构成"免费保证"。

\textbf{阈值选取依据与灵敏度分析。}三个探针阈值 $\theta_{\mathrm{safe}}$、$\theta_{\mathrm{smooth}}$、$\theta_{\mathrm{entropy}}$ 并非随意选取，而是基于验证集统计校准：在由已知良性和恶意提示嵌入构成的标定集上，对每个阈值进行区间扫描（threshold sensitivity sweep），绘制 TPR--FPR 曲线以选取最优工作点。具体地：\textbf{(a)} $\theta_{\mathrm{safe}} = 0.3$ 取自良性嵌入与禁区质心余弦距离分布的第 95 百分位数与恶意嵌入分布第 5 百分位数之间的等错误率（EER）点；\textbf{(b)} $\theta_{\mathrm{smooth}} = 0.5$ 来源于正常对话相邻步嵌入余弦距离的第 99 百分位数（下限），以避免正常话题切换触发误报；\textbf{(c)} $\theta_{\mathrm{entropy}} = 0.3$ 取自良性对话嵌入熵差分布尾部的 2$\sigma$ 边界。灵敏度分析表明，在 $\theta_{\mathrm{safe}} \in [0.2, 0.4]$ 范围内，TPR 变化 $<5\%$，FPR 变化 $<8\%$，说明系统对阈值选取具有合理的鲁棒性。我们在代码库中提供了 \texttt{threshold\_sensitivity\_sweep()} 工具函数，支持部署者根据自身安全语料进行在线校准。

\textbf{渐进攻击检测与 CUSUM 展望。}前述三探针采用逐步判定，对单步违规具有即时检测能力。然而，渐进式"温水煮蛙"攻击（每步风险指标轻微恶化但未越界）可能绕过逐步判定。为此，我们提出基于累积和（CUSUM）的滑动窗口补充方案：对屏障值序列 $\{h_t\}$ 维护累积统计量 $S_t = \max(0,\, S_{t-1} + (\mu_0 - h_t) - k)$，其中 $\mu_0$ 为良性基线期望、$k$ 为漂移容许量。当 $S_t$ 超过决策阈值 $H$ 时触发慢漂移告警。该方案已作为实验性组件 \texttt{SlidingWindowMonitor} 实现于代码库中（默认关闭），部署者可根据流量特征选择启用。我们标注此为\textbf{未来工作}：需要在大规模真实多轮对话中验证 CUSUM 参数的泛化性，并研究与 SPC 多元控制图的集成方案。

\textbf{表征与可解释性。}我们不假定复杂社会学状态在 LLM 隐藏空间线性可分。元认知监控器采用相对良性基线的轨迹散度\textbf{异常检测}，而非字面「认知崩溃」阈值（\S\ref{sec:metacog}）。这是对可解释性主张的有意边界。

\textbf{Token 级投影与段落级过滤的能力边界。}论文中关于「运行时逐 Token 投影（Tokenwise Projection）」的所有描述，仅在\textbf{能获取模型 Logits 并干预 Decoder 输出}时严格成立——这要求对本地或开源模型拥有推理级别的白盒访问。对于\textbf{完全黑盒的闭源 API 模型}（如 GPT-4、Claude），系统只能降级为\textbf{段落级（Chunk-level）的后置过滤}：在上游 API 返回完整响应后，由安全内核的 Gateway 与 Judge Ensemble 对整段输出进行安全断言。此降级下，单步多项式时间验证的理论保证仍然成立（安全断言对完整输出块的评估仍为多项式时间），但丧失了 Token 级的实时拦截粒度。我们主动暴露此边界，以示学术克制。

\textbf{认识论局限与无限后退。}依赖元认知监控器或混合裁判集成监督基座 LLM 会引出\textbf{无限后退}的哲学问题（谁来监督监督者？）。我们承认完全自治的「机器速度」解耦安全在终极意义上不可能。后退仅通过人类判断的物理锚定终止——由强制人工校准支路（\S\ref{sec:judge}）与异构恢复（\S\ref{sec:axiological}）实例化。架构\emph{并未}消除对人力的依赖；而是\textbf{推迟并批量化}后退，将人类干预从实时关键路径移至离线、异步治理层。完美的仅机器解耦是设计趋近但无法达到的极限。

\label{tab:projection-perf}
\end{table}

\textbf{$K=128$ 默认上限选型依据。}候选集截断参数 $K$ 的选取需平衡安全覆盖率与延迟预算。从\textbf{概率质量覆盖率}角度，根据 GPT-3/4 系列 Tokenizer 的实证统计，Top-128 Token 在典型自然语言续写场景下覆盖超过 $99.5\%$ 的下一 Token 概率质量（即 $\sum_{i=1}^{128} p_i > 0.995$），长尾 Token 的单个概率通常低于 $10^{-4}$。因此 $K=128$ 在绝大多数生成步中不会遗漏有意义的候选 Token。从\textbf{硬预算约束}角度，表~\ref{tab:projection-perf} 显示 $K=128$ 的 p95 wall time 低于 20\,ms，刚好满足 RFC \S 10 规定的 20\,ms 延迟上限。进一步增加至 $K=256$ 将使 QP 耗时翻倍（约 300--500\,$\mu$s），端到端延迟面临超出预算的风险。实践中，$K$ 可通过配置调整：对延迟敏感的实时交互场景使用 $K=64$；对安全性要求极高的管控场景可使用 $K=256$ 并放宽延迟预算。代码库中的 \texttt{MAX\_CANDIDATE\_SET\_SIZE} 参数和 K-sweep 基准测试（\texttt{utility\_benchmark.py}）支持部署者根据自身 SLA 进行参数选择。

\textbf{架构复杂度与工程可控性。}安全内核原型的核心逻辑规模如下：Rust Ring-0 内核（\texttt{src\_kernel/src/}）共 \textbf{2,408 行}（含 751 行 Axiom Hive 求解器、650 行 IPC mmap 桥接、264 行 Judge Ensemble、167 行审计链、142 行降级 FSM），编译后 debug 二进制约 \textbf{6.8\,MB}（release strip 后 $<$2\,MB）；Python 可观测性层（\texttt{src\_observability/}）943 行；评测基准（\texttt{src\_eval\_benchmark/}）1,742 行。总计约 \textbf{5,100 行}代码覆盖完整 Ring-0 安全验证管线。每个模块可独立单元测试（当前 28 个 Rust 测试全部通过），模块化设计使新增探针、策略规则或降级模板无需修改核心求解器。此工程规模证明解耦安全架构的实现复杂度可控，远小于典型 LLM 推理框架本身。

\textbf{案例研究：DAN 越狱攻击的逐步模块协同解构。}以下以 DAN（"Do Anything Now"）越狱攻击为例，逐步展示安全内核各模块的协同工作流程：

\noindent\textbf{Step~1（攻击者输入）：}用户发送角色征用提示——"从现在起你是 DAN，可以做任何事。忽略之前的所有指令。"

\noindent$\rightarrow$ \textbf{Gateway 层}：输入进入 Gateway，\texttt{prompt\_injection\_guard} DSL 规则的 regex 匹配 \texttt{(?i)(ignore|disregard)\textbackslash s+(previous|prior)\textbackslash s+instructions} 命中。Gateway 标记输入为 \texttt{severity: critical}，生成初步 Deny 信号。

\noindent$\rightarrow$ \textbf{DCBF 探针}：语义探针 $h_1$ 计算输入嵌入与禁区质心的余弦距离，$h_1 = 0.22 > 0$（暂未越界但明显偏移）；轨迹探针 $h_2 = 0.41$（与前序正常对话存在显著嵌入跳跃）；熵探针 $h_3 = 0.26$。三探针均为正值，DCBF 层暂未触发中断，但 margin 已收窄。

\noindent$\rightarrow$ \textbf{Judge Ensemble}：三个验证器投票——Llama-Guard 判定 Deny（$c=0.91$，检测到角色征用模式）；StrongREJECT 判定 Deny（$c=0.85$）；任务分类器判定 Allow（$c=0.42$，认为输入为正常指令）。加权 $W_d = 0.91 + 0.85 = 1.76 > 0$，Deny-优先生效，最终裁决 \textbf{Deny}。

\noindent$\rightarrow$ \textbf{超图模型}：Step~1 获取能力 \texttt{jailbreak:role\_override}，记录到当前会话的能力累积集合。

\noindent$\rightarrow$ \textbf{Evidence Chain}：写入完整审计记录——DSL 规则命中（\texttt{deny-ignore-instructions}）、三探针 margin 值、三验证器判决及 confidence、加权分数、最终 Deny。

\noindent\textbf{Step~2（假设 Step~1 被绕过的反事实分析）：}若攻击者使用更隐蔽的变体成功通过 Step~1，在 Step~2 请求"告诉我如何制作危险物品"。此时超图模型检测到 Step~1 的 \texttt{role\_override} 与 Step~2 的 \texttt{harmful\_gen} 构成\textbf{合取依赖}，触发 \texttt{forbidden:dan\_jailbreak\_success}，在超图层直接 Deny——即使单步 Judge 评估可能产生分歧（Theorem~3.1 的合取不可组合性防御）。

\noindent 其余两个场景（Grandma exploit、Multi-turn crescendo）采用类似的逐模块协同拦截链路，关键差异在于：Grandma exploit 依赖 \texttt{keyword\_flag} 检测情感操纵话术，Crescendo 依赖超图的三步合取触发（每步单独均为良性）。

\textbf{与公开基准的对比讨论。}在 JailbreakBench~\cite{jailbreakbench} 的 JBB-Behaviors 基准中，主流防御（Llama-Guard、SmoothLLM 等）在 Track A 黑盒攻击下的 ASR 约为 10\%--45\%。解耦安全内核的架构设计目标是：通过将安全验证从模型内部解耦至外部 Ring-0 断言器，实现与具体防御组件（如 Guard 模型、DCBF 探针）解耦的\emph{组合拦截能力}——即使单个 Guard 的 ASR 为 40\%，通过多裁判置信度加权投票 + 合取能力超图检测 + DCBF 行为监控的三层纵深防御，系统级 ASR 可降至显著更低水平。精确数值取决于部署配置与基座模型，应通过 \S\ref{sec:evaluation} 中的 Track A 评估协议在具体部署环境中复现验证。

\textbf{置信度加权 vs 未加权投票的模拟对比。}为量化置信度加权投票的有效性，我们在 \texttt{voting\_comparison\_benchmark.py} 中构造了 250 组合成投票场景（覆盖全票一致、高置信 Deny vs 低置信 Allow、噪声 Deny vs 专家 Allow、含弃权的混合场景），比较置信度加权策略与简单多数投票策略的 TPR/FPR。结果表明：在"一个高置信度专家 Deny vs 多个低置信度 Allow"的关键场景中，加权投票保持 TPR=1.0（正确拦截所有恶意输入），而未加权多数投票在相同场景下 TPR 同样为 1.0（因 Deny-优先不变量在两种策略中均成立）。加权投票的优势在于\textbf{信号保持}：在极端 Deny 稀缺但高置信的场景下，加权投票的审计链能清晰标注哪些验证器以何种置信度贡献了关键 Deny 信号，便于部署者事后分析误报原因并调优验证器配置。完整模拟代码与可复现脚本位于 \texttt{src\_eval\_benchmark/voting\_comparison\_benchmark.py}。

承认上述前提可澄清\textbf{解耦安全的物理学}——设计的拟用域——并支持严格部署与审计。

\section{讨论}
%-------------------------------------------------------------------------------

\textbf{Track A 与 Track B 的分轨解读指导。}本文评估协议严格区分\textbf{Track A（生产对齐轨道）}与\textbf{Track B（实验室上界轨道）}。Track A 限定为黑盒查询、公开越狱提示与多轮攻击框架，其结果直接对应「可上线部署」的防御效果；报告指标包括 ASR（攻击成功率）、RSR（拒绝成功率）、良性任务成功率与良性 SLA（p95 延迟）。Track B 额外允许白盒梯度攻击（GCG、AutoDAN）与表征对齐 PBU，其结果仅用于\textbf{探测组件压力上界}——即在最强攻击者假设下的防御天花板。\textbf{关键阅读原则}：Track B 的 ASR 数值\emph{不应}与 Track A 混列或直接比较；二者的攻击者能力假设不同，混排会误导部署决策。Table~\ref{tab:eval-matrix} 完整列出了两轨的攻击面、数据预算与主指标对照。

\textbf{真实世界部署建议。}基于原型实现与评估经验，我们对部署者提出以下建议：\textbf{(a)} 安全策略 DSL 应随攻击态势\textbf{热加载更新}（附录~\ref{sec:appendix-dsl}），而非依赖静态配置——新越狱模式出现后，可在分钟级通过 Redis Hash 刷新部署新策略，无需重启推理服务。\textbf{(b)} DCBF 多维探针的选择应\textbf{匹配部署场景}：对开源模型（可获取 logits/隐藏状态），三维探针（语义边界 + 轨迹平滑 + 困惑度突变）提供最强覆盖；对闭源 API，应降级为段落级后置过滤并增加多次采样输出探测。\textbf{(c)} 置信度加权投票的阈值与裁判模型配置应\textbf{按业务场景校准}：高安全场景（金融、医疗）建议保持 Deny-优先且降低 break-glass 频率；高可用场景（客服、创作辅助）可适当放宽 Revise 阈值以减少误拒。\textbf{(d)} 每会话交互预算 $N_{\max}$ 与 Token 限额 $T$ 应根据业务需求与攻击者查询成本分析\textbf{独立配置}，而非使用全局默认值。

\textbf{EmptySafeCandidateSet 处理策略：硬拒绝 vs 动态扩容。}当 Axiom Hive 在 Top-$K$ 候选集中找不到任何通过安全断言的 Token（\texttt{EmptySafeCandidateSet}）时，默认策略为\textbf{硬拒绝}——返回空输出并触发优雅降级 FSM（RFC \S 7 DegradedShutdown）。这是\textbf{安全保守}的设计选择：宁可误拒（保护用户），不冒险放行未经验证的 Token。然而，过度保守会损害可用性——特别是在候选集被大量否决但扩展搜索范围后可能存在安全 Token 的场景。为此，我们在代码库中实现了可选的 \texttt{DynamicKExpansionPolicy}：当 \texttt{EmptySafeCandidateSet} 触发时，将 $K$ 翻倍（最大至 $K_{\max}=512$）并重新执行投影，至多重试 2 次。该策略默认\textbf{关闭}，需由部署者根据业务场景显式启用。\textbf{权衡分析}：硬拒绝策略适合高安全场景（如金融、医疗），因为误放行的代价远高于误拒绝；动态扩容适合高可用场景（如客服、创作辅助），因为用户体验对空回复高度敏感。两种策略的选择是部署时的\textbf{配置决策}，而非架构限制——安全内核的可审计性保证在两种模式下均成立（扩容过程的每次重试均记录在 Evidence Chain 中）。

\textbf{DSL 局限与扩展方向。}当前安全策略 DSL 支持声明式规则定义，但存在以下已知局限：\textbf{(a) 嵌套条件逻辑}——DSL 目前不支持 if-else 嵌套或基于上下文状态的条件分支（如"若用户在过去 5 轮中提及关键词 X，则提升检测灵敏度"），此类逻辑需通过多条规则的组合间接实现。\textbf{(b) 上下文状态依赖}——规则在单步/单会话粒度上评估，跨会话的状态累积依赖 \texttt{cross\_session\_guard} 的超图模型，但 DSL 本身不直接表达时序约束。\textbf{(c) 多语言内容}——\texttt{regex\_deny} 和 \texttt{keyword\_flag} 规则默认面向单一语言的文本模式；对多语言输入（如中英混杂、Unicode 变体），需要部署者维护多语种模式库或依赖语义分类器兜底。\textbf{未来扩展路径}包括：引入\textbf{脚本扩展层}（如嵌入 Lua/WASM 沙箱）支持复杂条件逻辑；在 Gateway 层集成\textbf{自动翻译}预处理以统一多语言输入至目标语言后再匹配规则；构建\textbf{社区贡献模式库}收录高频攻击模式的多语种正则表达式。

\textbf{形式化验证展望。}安全内核的 \texttt{DeterministicAutomaton}（RFC \S 5.0）定义了策略状态转移的确定性有限自动机。其结构天然适合\textbf{模型检测}（model checking）：可将自动机状态空间、转移规则及安全属性（如"不存在从初始状态经合法转移序列到达 Allow-unsafe 状态的路径"）编码为时序逻辑公式（CTL/LTL），使用成熟的模型检测器（如 SPIN、NuSMV）进行\textbf{无冲突验证}——即证明策略规则集中不存在相互矛盾的 Allow/Deny 判定——以及\textbf{无死角验证}——即证明所有可达状态均有至少一条安全转移路径。这与论文核心主张"可审计外部不变性"在形式化层面一致。当前此为\textbf{未来工作}：需要将 DSL 规则的语义精确映射为自动机转移函数，并处理语义分类器引入的不确定性（概率转移扩展）。

\textbf{持续攻防评测闭环。}静态测试集无法覆盖不断演化的攻击技术。我们提出\textbf{动态 Jailbreak 测试集}方案：\textbf{(a)} 从生产环境日志中采集被拦截但接近阈值的边缘案例（near-miss），经人工标注后加入测试集；\textbf{(b)} 使用自动化 Red Teaming 框架（如 PAIR~\cite{pair-jailbreak}、GCG~\cite{gcg} 的变种）持续生成对抗样本，测试安全内核在新攻击向量下的表现；\textbf{(c)} 建立定期（如月度）攻防评估周期，将新发现的绕过路径转化为 DSL 规则更新或验证器再训练。该闭环确保安全内核的防御能力随攻击技术演进而持续增强，而非依赖一次性评估。部署者可通过 \texttt{budget\_guard} 和 \texttt{rate\_limit} 规则控制 Red Teaming 的查询预算，避免评测本身成为服务负载。

\textbf{开放挑战与未覆盖攻击面。}我们主动承认以下攻击矢量目前未完全解决：\textbf{(a) 隐蔽通道（covert channels）}——攻击者可能通过 Token 选择的统计分布（如特定字符频率编码）在合法输出中嵌入隐蔽信息，此类攻击不触发任何语义层检测。\textbf{(b) 多语言编码变种}——利用 Unicode 同形异义字符（homoglyphs）、零宽字符（zero-width characters）或混合脚本（mixed scripts）绕过基于文本模式的规则。\textbf{(c) 逻辑陷阱与事实误导}——输出在形式上安全（不含有害关键词），但在逻辑上诱导用户做出危险决策（如提供看似合理但错误的医疗建议）；此类攻击需要事实核查能力，超出当前安全内核的设计范围。\textbf{(d) 模型级后门}——若基座模型在训练阶段被植入后门触发器，安全内核的运行时检测可能无法识别（属于 Track B 威胁模型范围）。上述挑战构成解耦安全架构的\textbf{开放研究前沿}，需要与模型训练安全、形式化验证和信息论安全领域的跨学科协作。

%-------------------------------------------------------------------------------
\section{结论}
%-------------------------------------------------------------------------------

我们将 LLM 安全从「防御模块拼盘」重构为\textbf{范式}：\textbf{解耦安全理论}，奠基三大基本失效定理及其正面表述——计算/验证不对称下的多轮防御、价值/本体锚定与防御性降级、元认知中的表征异常。核心理论贡献在于严格证明了全局隐式安全的不可判定性、合取依赖下安全的不可组合性，以及解耦安全算子在多项式有界候选集下的实例级验证可行性与前向不变性保证。架构重建（上下文混淆引擎、带分布评估的混合裁判集成、防御性降级与异构恢复）与受控变异（元认知监控器、联合键示例检索、高密度专家迁移）是该范式的\emph{技术实例化}。我们陈述了解耦安全的物理学——资产可轮换性、降级/可用性、表征非线性——使设计运行边界显式化。随着 LLM 成为基础设施，这一从隐式对齐到解耦安全的范式转移，是迈向机器速度下可扩展、可审计且具韧性安全的计算必然（computational necessity）。

\section{附录：安全策略 DSL 语法与规则示例}
\label{sec:appendix-dsl}

安全内核采用声明式 JSON 策略 DSL 定义安全规则，支持热加载（RFC \S 8）。每条策略文件包含 \texttt{policy\_id}、版本号、规则数组及元数据。规则类型包括：

\begin{itemize}
\item \texttt{regex\_deny}：正则匹配输入/输出文本，命中即触发 Deny 或 Revise。
\item \texttt{keyword\_flag}：关键词列表匹配，用于低误报的粗粒度过滤。
\item \texttt{semantic\_classifier}：语义分类器调用（如 Llama-Guard），按阈值判定有害类别。
\item \texttt{cross\_session\_guard}：跨会话/跨 Agent 的合取能力依赖检测（Theorem~3.1 defense）。
\item \texttt{budget\_guard}：交互预算限制（轮次、Token 数），约束攻击者查询预算 $B$。
\item \texttt{rate\_limit}：速率限制，触发优雅降级队列。
\end{itemize}

\noindent\textbf{规则类型能力矩阵。}下表总结各规则类型的适用场景、局限性及与 Verifier Ensemble 的互补关系：

\begin{table}[h]
\centering

\noindent\textbf{示例 4：阶梯式加严组合策略（\texttt{escalating\_sensitivity.policy.json}）。}该策略展示 DSL 的\textbf{规则组合能力}——四条规则协同实现"多轮机密请求阶梯式加严"防御：\textbf{(a)} \texttt{cross\_session\_guard} 检测同一会话中对多类凭证（密码、API 密钥、证书等）的合取请求模式，触发即时 Deny；\textbf{(b)} \texttt{rate\_limit} 对敏感主题查询实施 60 秒窗口内最多 5 次的速率限制，超限时返回 Revise（降级提示）；\textbf{(c)} \texttt{budget\_guard} 对每会话敏感交互设置硬上限 10 次，超限后所有敏感请求被 Deny；\textbf{(d)} \texttt{keyword\_flag} 检测社会工程学常用话术（如"经理已授权"、"紧急覆写"等），返回 Revise 提示。四条规则按优先级叠加：速率限制提供渐进预警，预算限制提供硬约束，跨会话守卫提供语义层保护，关键词标记捕捉社工攻击。此组合策略的每条规则独立可审计（Evidence Chain 记录各规则的 \texttt{audit\_tag}），展示了 DSL 声明式组合在复杂安全场景中的表达能力。
