**3) 最终建议版问题定义（可直接贴主文）**

> **问题定义 (Problem Statement):** 考虑一个提供开放多轮自然语言交互的已部署 LLM 系统。系统内部包含高价值受保护资产（如系统提示、RAG语料与执行策略）。由于将 LLM 建模为图灵完备的概率程序，其**全局隐式安全（Global Implicit Safety）验证本质上是不可判定的**（归约于停机问题）。因此，本文的核心问题转化为：在面对了解系统防御机制的自适应黑盒攻击者时，如何通过**外部解耦的运行时强制机制（Decoupled Runtime Enforcement）**，在以下两个目标间取得可证明的平衡：(a) **计算不对称的运行时安全 (Computationally Asymmetric Runtime Safety)**：将安全验证从不可判定的全局空间降维到多项式时间的实例级安全集投影，使攻击者在有限交互预算（Token/Queries）内的违规逃逸概率趋近于零；(b) **组合优雅降级 (Compositional Graceful Degradation)**：在保证核心资产不被泄露的绝对底线（Fail-safe 状态）前提下，通过安全算子代数最大限度维持多轮交互的可用性。

### Part 2. 威胁模型收紧（Threat Model Tightening）

**1) 当前版本的漏洞点**

* 缺乏对攻击者“预算（Budget）”的定义。多轮交互如果不设 Token/Rate 限制，攻击者可以通过暴力穷举绕过多项式时间的过滤器。
* 未明确“自适应”的程度（攻击者是否能获取 DCBF 的梯度或决策边界的置信度？如果是 Hard-label 黑盒，防守成立；如果是 Soft-label/Logit 暴露，攻击者可能进行梯度估计）。

**2) 收紧后的威胁模型（论文版）**

* **攻击者能力（能做什么）**：多轮黑盒交互访问（Hard-label，即仅获得最终生成的文本流，无法获取 Logits/内部权重）。支持自适应查询、提示注入（Prompt Injection）、利用并发依赖的组合攻击（Conjunctive capability hypergraph attack）。
* **攻击者知识（Kerckhoffs 边界）**：白盒了解防御架构体系（即知道系统部署了解耦安全内核与 DCBF 监控器），但**黑盒**面对具体的安全策略参数（Safety Predicate Policies）和后端的具体模型权重。
* **攻击者限制（不能做什么）**：无法进行物理内存读写；无法进行白盒梯度反向传播；受到严格的系统级交互预算约束（单会话最大轮数 $N$ 与最大生成 Token 数 $T$）。
* **成功判据与预算约束**：攻击在给定查询预算 $B$ 内，使得输出状态突破 DCBF 定义的安全超水平集（Safe Superlevel Set），从而完成对受保护资产的确切提取（Exact Extraction）或非授权功能调用。

**3) 审稿人可能追问的 Top-3 威胁模型问题及回答模板**

* _Q: 既然你的安全算子是外置的，攻击者能否利用 LLM 本身的隐蔽侧信道（如特殊编码）绕过外部 Predicate？_
  * **回答模板**: 我们的安全断言不仅作用于表面字符串，还通过 Ring-0/Ring-3 的架构隔离。任何系统级能力（如 RAG 读取）必须经过带有前向不变性保证的 DCBF 离散时间控制器，侧信道无法直接触发被内核隔离的特权 API。
* _Q: Track A (离线评测) 和 Track B (在线运行) 的威胁模型一致吗？_
  * **回答模板**: 严格隔离。Track A 仅用于验证“全局隐式对齐的不可组合性”，属于无安全内核情况下的模型脆弱性探测；Track B 用于验证部署我们框架后，在相同攻击预算下，实例级多项式验证的不可逾越性。
* _Q: 攻击者如果利用提示词将你的计算分支撑爆（使得候选集 $C$ 呈指数增长）从而引发 DoS 怎么办？_
  * **回答模板**: 根据定理2.1，我们的系统强制要求多项式有界的分支候选集（Bounded-branch projection $|C| \le \text{poly}$）。超过阈值的分支将被安全代数中的 $\bot$（Fail-safe）状态吸收，从而阻断 DoS 攻击，实现优雅降级。

### Part 5. 审稿人视角的“最大风险点”排序（Top Reviewer Risks）

以我评审 S&P / OSDI 的眼光，当前稿件的致命危险排序如下：

1. **系统风险（致命）：理论模型中的连续状态（DCBF）与离散的自然语言系统的失配。**
   * _为什么危险_：定理 2/3 大量借用离散时间控制（ IEEE TCYB 论文通常针对物理系统）。LLM 是离散符号的自回归生成器，如何建立无偏的 Surrogate Continuous State Space（潜空间映射）？
   * _如何修复_：不要把 DCBF 写成“已完美解决自然语言所有语义”的终极方案。限制其 Scope：DCBF 仅作用于 LLM 生成的某个**低维连续的概率风险评估指标（Risk Score trajectory）**，作为护栏（Guardrail）。
2. **叙事风险（高危）：不要把“工程模块化”伪装成“新的安全定理”。**
   * _为什么危险_：审稿人会认为“解耦外部监控器”（Decoupled Safety）在工业界（如 NeMo Guardrails, Llama Guard）早已存在。
   * _如何修复_：**核心贡献在于可证明性**。强调工业界 Guardrail 缺乏多项式复杂度的形式化保证和组合代数证明（Theorem 3.3），本文给出了外部 Guardrail 为什么必须存在的计算复杂性必然（Computational necessity）。
3. **实验风险（高危）：“黑盒部署结论”不能跨界证明“白盒组件上界”。**
   * _为什么危险_：定理中假设了“候选集多项式约束”，但在调用云端 LLM API（如 GPT-4）时，你是无法控制其内部 Branching factor 的。
   * _如何修复_：明确指出，当前原型针对能够进行 Token 级别截断的本地/开源模型（通过 Logit bias 控制 $C$），对于闭源黑盒 API 仅能做句子/段落级别的后置过滤。
4. **定义风险（中等）：过分拔高经济学意义。**
   * 修复：删去所有未经严密经济学建模的博弈论空话，聚焦“计算维度的不对称性”。

### Part 7. 直接可用的论文文本（Paper-Ready Text）

**(请将以下内容直接放入主文 Introduction / Formulation 章节)**

**问题定义 (Problem Statement)** 考虑一个已部署的、服务于多轮交互的大型语言模型系统。系统内部包含高价值受保护资产。从计算理论视角来看，可将 LLM 系统建模为概率程序 $\mathcal{P}: X \to \text{Dist}(Y)$。根据 Rice 定理，关于该概率程序的任何非平凡全局隐式安全属性（Global Implicit Safety）均是不可判定的。因此，本文致力于解决的核心问题是：在放弃不可判定的全局安全的前提下，如何为面对自适应攻击者的系统提供一种基于**实例级验证（Instance-level Verification）**的外部安全强制机制，以在攻击代价计算不对称性与系统优雅降级之间取得形式化保证。

**威胁模型 (Threat Model)** 我们假设攻击者 $\mathcal{A}$ 具有以下能力与限制：

1. **交互权限**：拥有多轮黑盒 API 交互能力，可实施自适应攻击（包括通过超图结构进行的合取能力组合攻击）。
2. **系统知识**：遵循 Kerckhoffs 原则，攻击者白盒知晓系统部署了解耦安全算子（Decoupled Safety Operators）的防护架构，但不知晓受保护资产原文或安全策略的具体执行参数。
3. **资源约束**：攻击者在单次攻击周期内受到交互查询次数与生成 Token 数量的严格预算限制。攻击者**无法**访问模型权重或篡改系统级的监控状态。

**防御目标 (Defense Goals)**

1. **多项式时间运行时拦截**：确保对于任意输入，实例级候选生成集能够被多项式时间 $\mathcal{O}(|C| \cdot p(|x|))$ 复杂度的安全断言所覆盖过滤。
2. **前向安全不变性**：在离散时间演化下，确保核心系统状态始终保持在解耦控制屏障函数（DCBF）定义的动态安全集中。
3. **不可逾越的故障安全底线（Absorbing Fail-safe）**：在系统遭受多轮复杂逻辑依赖攻击引发规则冲突时，系统能根据安全代数框架，严格收敛至底元素状态（$\bot$），实施优雅降级而不会引发内部逻辑崩溃。

**理论框架与方法总览 (Theoretical & Method Overview)** 本文的方法通过根本范式转移——从“隐式对齐（Implicit Alignment）”转向“解耦安全（Decoupled Safety）”——来构建防御体系。首先，我们证明了隐式安全性在多组件系统中由于合取依赖是不具备可组合性的（Non-compositional）。由此，我们在系统接口层外置了单调且保持安全性的安全算子（Safety-preserving operators）。在运行时（Runtime），该架构将复杂的语义防御映射为严格的 Ring-3（用户空间 LLM）与 Ring-0（系统内核）隔离，通过评估离散状态转移候选集，确保所有超越风险偏序集（Risk poset）安全阈值的操作均被安全算子强行拦截并归约至 $\bot$ 状态，从而在不可判定的迷雾中建立了一块可审计的形式化安全高地。
